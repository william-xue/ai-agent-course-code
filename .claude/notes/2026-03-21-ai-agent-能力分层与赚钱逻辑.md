# AI Agent 能力分层与赚钱逻辑
> 2026-03-21 19:10 | ai-agent

## 代码真相
- LangChain = 大模型流水线框架，类比 Express
- 核心抽象全是已有概念换皮：pipe=Chain, middleware=Runnable, route handler=Tool, session=Memory, template engine=PromptTemplate
- 向量数据库（Milvus/Chroma）= RAG 的存储后端，Chroma≈SQLite，Milvus≈MySQL
- 技术本身门槛低，API 调用 + 链式组合 + Tool 定义都不难

## 设计决策
- AI Agent 的难点不在"会调 API"，在 Demo→生产的鸿沟
- 赚钱三要素：懂行业（AI 插哪个环节）、懂工程（稳定便宜快）、懂交付（包装成业务价值）

## 陷阱记录
- 通用 AI demo 不值钱，ChatGPT 已经是最好的通用聊天机器人
- 模型输出不确定性是最大工程挑战：乱码兜底、死循环烧钱、幻觉处理
- RAG 的核心不是向量库，是文档分块策略——切错了检索全废

## 私人笔记
能力分层：
- 初级：会调 API 串 Chain（入场券，不值钱）
- 中级（能接单）：Prompt 工程 + 兜底设计 + 分块策略
- 高级（能赚大钱）：行业建模 + 评估体系 + 成本控制 + 多 Agent 协调

价值公式：AI 应用价值 = 行业知识壁垒 × 数据私有性 × 重复劳动量

近钱场景：企业 RAG 知识库（2-10万）、法律合同审查、电商客服、财务自动化
技术是入场券不是护城河，找一个熟悉的行业痛点用这套工具解决才是正路
